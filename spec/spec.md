“AI 재고·수요 예측 운영툴” 

(수요 예측 → 발주 추천 → 이상징후 탐지)

# 만들 제품

판매/재고/입고/반품 데이터를 모아서

SKU 단위로 “다음 7/14/30일 수요 예측”

“재고 소진 예상일” + “발주 추천 수량” + “품절 위험 알림”

프로모션(할인/광고) 영향도 반영 옵션

# ETL / 데이터 파이프라인 핵심

소스: 주문/결제, 재고, 상품, 프로모션, 광고 집행(가짜 데이터라도 스키마는 실무처럼)

배치 ETL: Raw → Staging → Mart(집계 테이블)

특징(Feature) 생성: 요일/시즌, 가격변동, 할인율, 리드타임, 재고회전율, 반품률

모델 학습/예측 결과를 “Prediction 테이블”로 적재

실시간성 옵션: 주문 이벤트 들어오면 간단한 집계만 스트리밍 업데이트

# AI 포인트

수요 예측(기본: XGBoost/LightGBM or Prophet, 나중: LSTM/Temporal Fusion Transformer는 “확장”으로)

이상탐지: 갑자기 판매 급증/급감, 반품률 폭증, 특정 셀러/상품 이슈

“왜 이런 추천이 나왔는지” 설명(특징 기여도/룰 기반 설명)

# Admin 기능

SKU 리스트, 필터/정렬, “품절 위험 TOP”

발주안 생성/승인/상태관리(워크플로우)

알림 설정(슬랙/이메일은 모킹 가능)

권한(운영자/매니저/관리자), 감사로그

# 기술 스택 예시

FE: React + AntD(테이블/필터 강함)

BE: Node(NestJS/Express) + PostgreSQL

ETL: Python(추천) + Airflow/Prefect(로컬도 가능)

모델: Python ML + 결과만 API로 서빙(또는 배치 저장)
